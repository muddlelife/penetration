1.Cookie和Session的区别
工作原理
    1.cookie
        1.浏览器第一次发送请求到服务器端
        2.服务器端创建cookie，cookie中包含用户的信息，然后将cookie发送到浏览器端
        3.浏览器再次访问服务器端时会携带服务器端创建的cookie
        4.服务器端通过cookie中携带的数据区分不同的用户
    2.session
        1.浏览器第一次发送请求到服务器端，服务器端创建一个session，同时创建一个特殊的cookie，然后将cookie发送至浏览器端
        2.浏览器发送第N（N>1）次请求到服务器端，浏览器端访问服务器端就会携带该name为jsessionid的cookie对象
        3.服务器端根据name为jsessionid的cookie的alue（sessionld），去查询session对象，从而区分不同用户
        4.补充
            1.name为jsessionid的cookie不存在（关闭或更换浏览器），返回1中重新创建session与特殊的cookie
            2.name为JSESSIONID的Cookie存在，根据value中的SessionId去寻找session对象，value为SessionId不存在（Session对象默认存活30分钟），返回1中重新去创建Session与特殊的Cookie
            3.value为sessionid存在，返回session对象
相同点
    cookie和session都是用来跟踪浏览器用户身份的会话方式
不同点
    1.cookie数据存放在客户的浏览器上，session数据放在服务器上
    2.cookie不是很安全，别人可以分析存放在本地的cookie进行cookie欺骗，要考虑安全应session
    3.session会在一定时间内保存在服务器上，当访问增多，会占用服务器性能，要减轻服务器性能，用cookie
    4.cookie在保存数据时要注意个数和大小的限制，大小一般是4k
    5.将登陆信息等重要信息存放在session，其他信息如果需要，放在cookie

2.MIME类型
    媒体类型（MIME）是一种标准，用来表示文档、文件或字节流的性质和格式
    通用结构 type/subtype
        type：表示可以被分多个子类的独立类别
        subtype:表示细分后的每个类型
    对大小写不敏感，传统写法为小写
    独立类型：
        test(表明是普通文本，人类可读):text/plain, text/html, test/css, test/javascript
        image(某中图像，不包括食品，但是动态图gif之类也用):image/git, image/png, image/jpeg, image/bmp, image/webp, image/x-icon, image/vnd.microsoft.icon
        audio(某种音频文件):audio/midi, audio/mpeg, audio/webm, audio/ogg, audio/wav
        video(某种视频文件):video/webm, video/ogg
        application(某种二进制数据):application/octet-stream, application/pkcs12, application/vnd.mspowerpoint, application/xhtml+xml, application/xml,  application/pdf

3.哈希算法
哈希算法也叫摘要算法，作用就是对一组数据进行计算，得到一个固定长度的输出摘要
特点
    1.相同的输入一定得到相同的输出
    2.不同的输入大概率得到不同的输出

哈希算法的目的就是为了验证原始数据是否被篡改

哈希碰撞
    两个不同的输入得到了相同的输出，所以一个安全的哈希算法必须满足
        1.碰撞概率低
        2.不能猜测输出（就是不能从输出推算出输入，看不到任何规律）
常见的哈希算法
    算法    输出长度    输出长度（字节）
    MD5     128bits     16bytes
    SHA-1   160bits     20bytes
    SHA-256 256bits     32bytes
    SHA-512 512bits     64bytes
根据碰撞概率，哈希算法的输出长度越长，就越难产生碰撞，也就越安全
用途
    1.验证下载文件的原始性
        通过网站提供的文件hash值，和自己下载后的文件hash值进行对比，如果一致则说明是同一个文件
    2.存储用户口令
        一般数据库用户的密码都是密文，也就是通过MD5加密（或者是其他的hash算法）后的hash值，这样不仅数据库的管理员不可以看到用户的原始口令，就算是数据库泄露，其他人也不能看到用户的密码，而用户登陆的时候输入自己的口令，程序会根据用户的用户名从数据库找到对应的hash值与用户刚输入口令的hash值进行对比，如果一致，说明是同一个用户。
彩虹表
    因为不可能穷举所有的数据得到hash值来和获得的hash值对比，因此需要有一个常用口令和hash的对照表，如果用户的口令是一些常见的口令，与彩虹表进行对照，就有可能查到原始口令，因此不要使用常用密码
加盐
    对每个口令额外的添加随机数，称为加盐
    可以抵御彩虹表攻击，可以在数据库用户账号列表多生成一列，这一列的数据由系统随机生成，当用户第一次注册账户时，系统会随机生成一个盐值，把盐值和用户口令进行组合，得到一个加盐后的口令，然后进行hash，再将得到的hash值进行存储，当然盐值也会存储。这样，每当用户登陆时，系统都会用盐值和口令进行hash，再与存储的hash值对比，就算数据库泄露，黑客也不能通过数据库中的hash值进行彩虹表攻击。

4.web服务器、中间件和web容器区别
web服务器
    一个web服务器是服务器软件，或者一个或多个系统的计算机专用于运行该软件，满足客户端的HTTP请求，web服务器可以管理客户端的http请求web资源与一个或多个他的配置/服务的网站
    常见web服务器
        ISS、Apache、nginx、tomcat、webloginc、websphere
中间件
    中间件是介于操作系统和在其上运行的应用程序之间的软件。中间件充当的是隐藏转换层，实现了分布式应用程序的通信和数据管理，因为它将两个应用程序连接在一起，使数据和数据库在“中间件”轻松传递，也称为管道，通过中间件，用户可执行很多请求，比如在web浏览器上提交表单，或者允许web服务器基于用户的配置文件返回动态网页
    常见中间件包括
        数据库中间件
        应用程序服务器中间件
        面小消息的中间件
        web中间件和事务处理监视器
    web中间件
        常见的web中间件
            Tomcat(tomcat可以单独运行，也可以被集成在apache中，apache处理静态文件比tomcat快)
            Webloginc
            Apache
web容器
    处理http的容器，例如tomcat（拥有JSP容器，servlet容器），ISS（拥有ASP容器）。web容器用于给处于其中的应用程序组件（JSP，servlet）提供一个环境，是中间件的一个组成成分，实现了对动态语言的解析，例如tomcat可以解析jsp，因为其内部有个jsp容器
    常见web容器
        JSP容器
        SERVLET容器
        ASP容器

5.CVE漏洞
    CVE的英文全称是“Common Vulnerabilities & Exposures”公共漏洞和暴露。CVE将广泛认同的信息安全漏洞或者已经暴露出来的弱点给一个公共的名称。
    格式
        CVE-YYYY-NNNN（4个数字）
        CVE-YYYY-id（至少4个数字，可以递增到无限）
        如CVE-2014-1984，或者CVE-2014-3921002    

6.DNS域名解析
    DNS,就是域名系统，是互联网上作为域名和IP地址相互映射的一个分布式数据库。DNS能够使用户更方便的访问互联网，而不用去记住能够被机器直接读取的IP地址，通过域名，最终得到该域名对应的IP地址的过程叫做域名解析（或主机名解析）
    流程
        1.本地hosts文件
            windows 默认 c:\windows\System32\drivers\etc\hosts
            linux 默认 /etc/hosts
        2.本地DNS缓存
        3.本地DNS服务器
        4.DNS服务器查找
    DNS迭代和递归区别
        1.迭代
            本地服务器向根DNS服务器发出请求，而根DNS服务器只是给出下一级DNS服务器地址，然后本地DNS服务器再向下一级DNS发送查询请求直至得到最终IP
        具体流程
            1.在hosts静态文件、DNS解析器缓存中查找某主机的ip地址

            2.上一步无法找到，在DNS本地服务器（即域服务器）查找所有本层次的区域服务器

            3.本地DNS服务器查不到就查询上一层次的所有区域服务器，以此类推直至根域名DNS服务器

            4.到达根域名服务器后又向下查询，直至查到结果为止。
        2.递归
            用户只向本地DNS服务器发出请求，然后得到肯定或否定答案
        具体流程
            1.在hosts静态文件、DNS解析器缓存中查找某主机的ip地址

            2.上一步无法找到，去DNS本地服务器（即域服务器）查找，其本质是去区域服务器、服务器缓存中查找

            3.本地DNS服务器查不到就根据‘根提示文件’向负责顶级域‘.com’的DNS服务器查询

            4.根DNS服务器根据查询域名中的‘xyz.com’，再向xyz.com的区域服务器查询

            5.www.xyz.abc.com的DNS服务器直接解析该域名，将查询到的ip再原路返回给请求查询的主机

8.Web程序三层架构
    通常意义上将整个业务应用划分为界面层+业务逻辑层+数据访问层
    用户访问网页过程
        1.Web浏览器中输入网址并连接目标服务器
        2.业务罗基层的web服务器从本地加载index.php脚本并解析
        3.脚本连接位于数据访问层的DBMS，并执行SQL
        4.数据访问层的DBMS返回SQL的执行结果给web server
        5.业务逻辑层的web server 将页面封装成HTML格式发送给表示层的浏览器
        6.表示层的浏览器解析HTML并将内容呈现给用户

9.ASP、PHP、JSP的比较
    1.ASP
        ASP采用脚本语言VBScript（javascript）作为自己的开发语言。ASP是Microssoft开发的动态网页语言，只能在微软的服务器产品IIS上执行
    2.PHP
        PHP是跨平台的服务器端的嵌入式脚本语言，它几乎都要借用C、Java和Perl语言的语法，同时结合PHP自己的特性，使得Web开发者能够快速地写出动态页面。PHP特点是：支持绝大多数数据库，并且其源码是完全公开的
        PHP可在为indows、unix、linux的web服务器上正常执行，还支持IIS、Apache等一般的Web服务器，用户更换平台时，无需变换PHP代码
        PHP与MySQL是目前绝佳组合，用户可以自己编写外围的函数的函数间接存取数据库，通过这个的途径，在更换使用的数据库时，可以轻松地修改编码以适应这样的变化
    3.JSP
        JSP同PHP类似，几乎可以在所有平台上执行，如windows、linux、Unix。web服务器Apache已经能够支持JSP，而Apache广泛应用在windows、Unix和linux上，因此JSP有更广泛的执行平台
        从一个平台移植到另外一个平台时，JSP和javaBean甚至不用重新编译，因为java字节码都是标准的，与平台无关。ASP、PHP、JSP三者都是面向web服务器的技术，客户端浏览器不需要任何附加的软甲支持
        普通的HTML页面只依赖于Web服务器，但ASP、PHP、JSP页面需要附加的语言引擎分析和执行程序代码。程序代码的执行结果被重新嵌入HTML代码中，然后一起发送给浏览器。三者都提高了在HTML代码中混合某种程序代码、由语言引擎解释执行程序代码的能力。JSP代码被编译成Servlet并由java虚拟机解释执行，这种编译操作仅在对JSP页面的第一次请求时发生
        在ASP、PHP、JSP环境下，HTML代码主要负责描述信息的显示样式，而程序代码则用来描述处理逻辑
