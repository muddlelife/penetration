# Google Hack

Google是世界上最强大的搜索引擎，而它之所以强大的一个重要的原因就是它的Google语法

Google语法分为基础语法和高级语法，我们常常用高级语法来搜索一些敏感信息

## 基本语法（日常使用中的搜索方法）

1. 默认是“and”逻辑

   比如说输入“猫”和“狗”关键字，搜索结果就会包含有“猫”和“狗”的内容，注意：“猫”和“狗”之间要加空

2. "OR"

   注意要**大写**，因为默认是“and”，所以你在搜索“或”的时候一定要在关键词中间加一个“OR”。如 猫 OR 狗。搜索结果是两个之一

3. "-"

   如果只想看到的搜索结果中不要包含其他的关键字，你可以在要搜索的关键字后面加“-”再接关键字。如想看到猫而不想看到狗：“猫 -狗”

4. 忽略关键词

   输入关键词的时候，Chrome会选择性的忽略一些它认为不重要的词，比如输入“www.baidu.com”的时候，它会忽略“www”，但是你不想忽略它的时候，你可以在它前面加上“+”，如“+www”

5. 模糊搜索*

   强制结果包含此关键字

6. ”关键字"

   在关键字上加上双引号，强制搜索结果出现此关键字

## 高级语法（hacker 常用，与基础语法结合，前五个最常用）

**首先先明确**

* inurl指的是网站的url
* intext指的是网站的内容
* intitle指的是网站的标题

1. "site:"将搜索内容限制在站点域名之内

   ```python
   "web安全" site:zhihu.com
   "sql注入" site:csdn.net
   "图片" site:pan.baidu.com
   ```

2. "filetype:"指搜索指定后缀的文件（黑客领域常用）

   ```python
   filetype:mdb  #会显示一些网络的数据库文件
   "犯罪心理学"filetype:pdf
   "身份证" filetype:xls
   filetype:pdf site:www.pan.baidu.com
   ```

3. "inurl:"会将搜索范围限制在URL或者网站的页面上，在黑客搜索中用处很大

   ```python
   inurl:.php?id=
   inurl:view.php=?
   inurl:.jsp?id=
   inurl:.asp?id=
   inurl:/admin/login.php  # 搜索网址中有”/admin/login.php”的网页
   inurl:login  # 搜索网址中有"login"等登录网页
   ```

4. "intitle:"搜索标题存在特定关键字的网页

   ```python
   intitle:后台登录
   intitle:后台管理 filetype:php
   intitle:index of "keyword"
   intitle:index of "parent directory"  # 搜索根目录相关的索引信息
   intitle:index of "password"  # 搜索密码相关的索引目录信息
   intitle:index of "login"  # 搜索登录页面信息
   intitle:index of "admin"  # 搜索后台管理页面信息
   ```

5. "intext:"搜索网页正文内容中存在特定关键字的网页，忽略网页中的超文本链接、URL和题目

   ```python
   intext:Powered by Discuz  # 搜索Discuz论坛相关的页面
   intext:Powered by wordpress  # 搜索wordpress制作的博客网址
   intext:Powered by *CMS  # 搜索*CMS相关的页面
   intext:Powered by xxx inurl:login  # 搜索此类网址的后台登录页面
   ```

6. "nanchor:"在页面的链接锚点进行搜索。链接锚点指的是一个链接的描述文本

   ```python
   inanchor:login
   inanchor:点击这里
   ```

7. "link:"（链接）

   查询所有链接到某个特定URL上的列表，注意是特定的URL页面

   ```python
   link:www.baidu.com
   ```

8. "cache:"（缓存）

   帮助我们查找到Google索引过的页面副本。最大的好处就是即使源文件界面不存在了

   ```python
   cache:www.baidu.com
   ```

9. "related:"（相关的）

   用于搜索出页面相关的网站的，可以辅助我们搜索同类的页面

   ```python
   related:www.google.com  # 可显示一些其他类的搜索引擎
   ```

10. "info:"

    可以搜索到关于一个URL的更多信息的页面列表，包括网页的cache，还有与这个网页相关的网页等等

    ```python
    info:www.google.com 
    ```

11. "author:"

    返回的是Google论坛里特定作者的新闻文章。
